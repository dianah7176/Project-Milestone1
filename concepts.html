Key Concepts

These terms explain how modern web communication, security, and performance work together.

Statelessness
Each request stands alone with no memory of previous interactions. Websites use cookies, sessions, or tokens to track users.
Persistent Connection
One connection handles multiple requests and responses to reduce setup delays. HTTP/1.1 introduced this as the default behavior.
Multiplexing
Multiple data streams share one connection at the same time. HTTP/2 and HTTP/3 use multiplexing to speed up page loads.
Head-of-Line Blocking
A single delayed packet blocks other packets behind it on a TCP connection. QUIC avoids this issue by handling streams independently.
TLS (Transport Layer Security)
Security protocol that encrypts web traffic and protects it from tampering and impersonation. HTTPS relies on TLS.
QUIC
A UDP-based transport protocol that integrates TLS 1.3 and improves speed, security, and mobility. It powers HTTP/3.
Caching
Stores copies of responses to speed up future requests. Common cache headers include Cache-Control, ETag, and validation rules.
Request–Response Cycle
The process where a browser sends a request to a server and the server sends back a response with content and a status code.
Digital Certificate
An electronic document that verifies a website’s identity. Certificate Authorities issue certificates for HTTPS encryption.
Latency
The time it takes for data to travel from the client to the server and back. Lower latency improves user experience.
