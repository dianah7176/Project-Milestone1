Key Concepts

Use these terms while you study HTTP versions and HTTPS. Each definition stays short and practical.

Statelessness
Each request stands alone. Apps store user state with cookies, sessions, or tokens.
Persistent Connection
One TCP connection serves multiple requests and responses to reduce setup delay. HTTP/1.1 made this the default.
Multiplexing
One connection carries many parallel streams to cut wait time. HTTP/2 and HTTP/3 use it.
Head-of-Line Blocking
One stalled packet blocks others on the same TCP connection. QUIC avoids this by recovering streams independently.
TLS (Transport Layer Security)
Security layer that adds confidentiality, integrity, and authentication to web traffic. HTTPS uses TLS.
QUIC
UDP-based transport that integrates TLS 1.3 and stream-level recovery. It powers HTTP/3 for lower latency and better mobility.
Caching
Reuse responses to speed repeat loads. Common headers include Cache-Control, ETag, and validators.
